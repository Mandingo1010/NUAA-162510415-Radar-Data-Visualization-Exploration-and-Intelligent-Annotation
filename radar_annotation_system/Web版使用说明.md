# 🌐 雷达标注系统 - Web版使用说明

## 🎯 系统概述

这是一个基于Web的雷达数据可视化与智能标注系统，支持文件夹拖拽上传、自动文件分类、智能标注和人工审核功能。

## 🚀 快速启动

### 方法1：一键启动（推荐）
```bash
# 进入项目目录
cd D:\desktop\test\radar_annotation_system\web_interface

# 双击运行
start_web.py
```

### 方法2：手动启动
```bash
# 1. 安装依赖（如果还没有安装）
pip install flask opencv-python pillow numpy

# 2. 启动应用
python app.py
```

### 启动成功标志
看到以下信息说明启动成功：
```
🚀 启动Web应用...
✅ 浏览器已打开: http://localhost:5000
```

## 📋 界面介绍

### 主要区域
1. **左侧区域** - 文件上传和管理
2. **中间区域** - 图像显示和标注
3. **右侧区域** - 人工审核面板

### 功能模块
- **文件上传** - 支持拖拽文件夹
- **自动分类** - 智能识别文件类型
- **智能标注** - AI自动检测目标
- **人工审核** - 通过/拒绝标注
- **结果导出** - 下载标注结果

## 🎮 使用流程

### 第1步：上传文件
1. **拖拽上传**：直接拖拽文件夹到上传区域
2. **点击选择**：点击"选择文件夹"按钮
3. **自动分类**：系统自动识别雷达、图像、配置文件

### 第2步：开始处理
1. 选择处理模式（推荐"自动处理"）
2. 点击"开始处理"按钮
3. 等待AI完成智能标注

### 第3步：人工审核
1. **查看标注**：在图像上查看AI生成的标注框
2. **选择标注**：点击标注框查看详情
3. **审核操作**：
   - 绿色"通过"按钮 - 确认标注正确
   - 红色"拒绝"按钮 - 标记标注错误
   - 黄色"编辑"按钮 - 修改标注信息

### 第4步：批量操作
1. **全部通过** - 通过所有待审核标注
2. **全部拒绝** - 拒绝所有待审核标注
3. **导出结果** - 下载审核结果

- **A** - 通过当前标注
- **D** - 拒绝当前标注

- **→** - 下一个标注

## 🚨 故障排除

### 常见问题

**Q: 浏览器无法访问？**
A:
1. 检查命令提示符是否显示启动成功
2. 手动访问 http://localhost:5000
3. 检查防火墙设置
4. 尝试更换端口（修改app.py中的端口号）

**Q: 文件上传失败？**
A:
1. 检查文件大小是否超过1GB
2. 确保文件格式被支持
3. 重新启动应用
4. 检查上传权限

**Q: 处理过程很慢？**
A:
1. 大文件需要更长处理时间
2. 检查网络连接
3. 关闭其他占用资源的程序
4. 减少同时处理的文件数量

**Q: 标注不准确？**
A:
1. AI检测结果可能有误差，需要人工审核
2. 使用"编辑"功能修正标注
3. 检查图像质量是否清晰
4. 确认文件内容是否为相关场景

### 错误代码
- **404** - 页面不存在
- **413** - 文件太大
- **500** - 服务器内部错误
